## Why I built it
Being a Dane living abroad makes it hard to watch Danish TV. I did not like using commercial VPNs because of excessive tracking and having one more monthly plan. Additionally, some streaming providers block IP ranges from known VPN machines, and I don't want to be part of this whack-a-mole-just-switch-to-a-different-IP game. I need something that just works.

## Hardware and goal
We really only need two pieces of hardware, total price: ~75 Euros. If you include the optional parts, it's about ~100 Euros

1. Raspberry Pi 4, Price: 50-60 Euros
  - A Raspberry Pi 3 (25-30 Euros) is also fine, but at the time of writing, the latest Raspberry Pi is 5, so Raspberry Pi 3 parts are not as easy to find. I just went for the 4 because it was plenty of power and easy to find parts.
2. MicroSD card. Anything over 16 GB is probaby fine. I got a 32 GB for 13 Euros
3. [Optional] PoE (Power over ethernet) HAT, Price: 20-25 Euros
4. [Optional] Housing/case for the Raspberry Pi, Price ~5 Euros

Other very useful hardware:
- A USB keyboard and mouse, to control the Raspberry Pi during setup. At the very least, a USB keyboard. The Raspberry Pi does have Bluetooth, but until you get your Bluetooth keyboard or mouse connected, it's a little bit hard to really do much. There are probably some sophisticated tricks to still control the Pi if you really want to .
- A Micro HDMI cable to see the Raspberry Pi interface on a display. You can also connect to it remotely from your usual laptop or desktop, but again, this €10 cable just makes it a lot easier. And then you have one more cable in your cable drawer. You're welcome!

Goal: Use the [Tailscale](https://tailscale.com/docs/install/appletv) app on my Apple TV to connect to my own VPN in Denmark. No getting out of the couch. I just want to use the remote!

## Plan
1. Buy a Raspberry Pi - a small cheap computer.
2. Install Tailscale on the Raspberry Pi. Configure and test it.
3. Send the Raspberry Pi to my brother-in-law in Denmark.

Tailscale is an awesome service that allows you to create your own personal network (tailnet) so that devices that are located anywhere can see each other and talk to each other without going through a central server. You just install the Tailscale app or daemon on your devices, and all devices can all talk directly to each other - no slow down, no middleman.

My brother-in-law lives in Denmark, he has a symmetric 1 Gb/s up and down connection, and he doesn't mind sharing up to 50 Mb/s with me. Additionally, it's always helpful that he's kind of a nerd like me, should the Raspberry Pi ever need some physical TLC!

## 2. Install Tailscale
Tailscale was the part that removed most of the traditional VPN pain.

``` sh
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up --ssh
```

That gave me two things right away:

1. The Raspberry Pi joined my tailnet
1. I could SSH to it from my laptop or even an SSH app on my phone, even when they're not on the same network, because they are once they join the Tailscale tailnet network

## 3. Turn the Pi into an exit node
To make the Pi behave like a VPN endpoint for full internet traffic, you need to toggle "Exit Node" _on_ for the device in the Tailscale admin panel. You could also do this directly from the command line on the Raspberry Pi with a command like:

```bash
sudo tailscale up --advertise-exit-node --ssh
```

This "advertise as exit node" is important in this setup: the Pi is not just another node/device on the tailnet. It becomes the machine other devices can use as their internet egress (exit node), so that the traffic from those other nodes can go through the Pi, should they choose to do so.

Each device on Tailscale gets an authentication key that it uses to authenticate itself on the tailnet. By default, this key expires after a few months. I wouldn't risk that this key expires since I didn't have direct access to the Pi. I turned off key-expiration for the PI in the Tailscale Admin Panel. As long as I can SSH into the Pi remotely, I can obviously refresh the key. If for some reason I should forget to do that, or the Pi goes offline for a long time for whatever reason and then comes up after being expired, I still can't connect to it if the key is expired. So, turn off key expiration for the Pi in the Tailscale admin panel

## 4. Test it, test it
Since this was going to be in another country, I had to make sure that it _just worked_. As I mentioned above, I decided to be powered over the Ethernet cable as mentioned above. I obviously tried to unplug and replug, unplug and replug many times to see if it would automatically boot up, join the Tailscale network, and advertise itself as an exit node.


I left it on the table connected for about a week just to see if it would go to sleep or shut down or something. And then let's try to do the same again: unplug and replug. It kept coming back online.


So I shipped it to Denmark!

## A few tips

- Prefer wired Ethernet over Wi-Fi: It's just a bit more reliable.
- If you don't power over the ethernet cable, use a good USB-C power adapter. Lately, I've been running a Raspberry Pi 5 at home (so that I can see Austrian TV when abroad and test Openclaw), and this Pi would repeatedly shut down when just getting power off my router's USB-port

## Final thought
This was one of those projects where the best part was how little custom work I had to do. The Raspberry Pi gave me a cheap always-on machine, and Tailscale gave me the private network and exit-node plumbing without turning it into a networking hobby project.

For a home VPN, that was exactly the right tradeoff.
